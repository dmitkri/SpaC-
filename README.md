# Управление СПА Салоном

Проект по С++. Управление СПА салоном. 5 семестр.

## Версия для GitHub Pages

Этот проект адаптирован для работы на GitHub Pages без сервера. Используется mock API с хранением данных в localStorage браузера.

### Как опубликовать на GitHub Pages

1. Перейдите в настройки репозитория на GitHub: https://github.com/dmitkri/SpaC-/settings/pages
2. В разделе "Build and deployment":
   - **Source**: выберите "Deploy from a branch"
   - **Branch**: выберите `main` и папку `/ (root)`
3. Нажмите **Save**
4. Через несколько минут ваш сайт будет доступен по адресу: 
   `https://dmitkri.github.io/SpaC-/`

**Важно:** Используйте обычный статический хостинг (Deploy from a branch), а не GitHub Actions!

### Структура файлов

- **HTML файлы** - страницы приложения:
  - `home.html` - главная страница
  - `index.html` - панель управления (для администраторов)
  - `login.html` - вход и регистрация
  - `client.html` - онлайн запись для клиентов
  - `booking.html` - форма записи
  - `client-dashboard.html` - личный кабинет клиента

- **JavaScript файлы**:
  - `mock-api.js` - эмуляция API сервера (использует localStorage)
  - `script.js` - логика панели управления
  - `client.js` - логика клиентской части
  - `booking.js` - логика формы записи

- **Другие файлы**:
  - `style.css` - стили приложения
  - `.nojekyll` - отключает обработку Jekyll на GitHub Pages
  - `.gitignore` - исключает C++ файлы из репозитория

### Особенности версии для GitHub Pages

- Все данные хранятся в localStorage браузера
- Не требуется сервер - приложение полностью клиентское
- Данные сохраняются локально в браузере пользователя
- Все функции API эмулируются через mock-api.js

### Вход в админ-панель

По умолчанию создан администратор для входа:
- **Email:** `admin@spa.ru`
- **Пароль:** `admin123`
- **Роль:** Суперадминистратор (доступ ко всем функциям)

1. Откройте страницу входа: `login.html`
2. Введите email и пароль администратора
3. После входа вы будете перенаправлены в админ-панель (`index.html`)

**Примечание:** При первом открытии сайта создается администратор автоматически. Данные хранятся в localStorage браузера.

### Синхронизация данных между устройствами

По умолчанию данные хранятся локально в браузере каждого устройства. Чтобы видеть записи с телефона на компьютере и наоборот, нужно настроить синхронизацию.

#### Быстрая настройка через JSONBin.io (Рекомендуется)

1. Перейдите в админ-панели во вкладку **"Синхронизация"**
2. Зарегистрируйтесь на [jsonbin.io](https://jsonbin.io) (бесплатно)
3. Создайте новый API ключ в личном кабинете
4. Скопируйте Master Key (начинается с `$2b$10$...`)
5. Вставьте ключ в форму "JSONBin.io" на странице настроек
6. Нажмите "Включить синхронизацию JSONBin"

После этого все данные будут автоматически синхронизироваться между всеми вашими устройствами!

#### Альтернатива: Firebase

Для более продвинутой синхронизации можно использовать Firebase:

1. Создайте проект на [Firebase Console](https://console.firebase.google.com)
2. Создайте Realtime Database
3. Включите анонимную аутентификацию (или настройте правила безопасности)
4. Скопируйте конфигурацию проекта
5. Вставьте данные в форму "Firebase" на странице настроек

#### Резервное копирование

На странице настроек синхронизации доступна функция **"Экспортировать данные"** для создания резервной копии всех данных в JSON файле.

### Оригинальная версия (с C++ сервером)

Если нужно запустить версию с C++ сервером:
- Скомпилируйте сервер: `make`
- Запустите: `./spa_server`
- Откройте в браузере: `http://localhost:8081`
